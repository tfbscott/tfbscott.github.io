System Programming

I learned the benefit of being able to rapidly test and update game projects very early in my DigiPen career. For Cleanup on Aisle 6.67, the levels were loaded from a text file in 14x14 chunks. Each level could feature multiple of these chunks. Since the files were loaded at runtime, we could change the levels without needing to reboot the game.

EFO: Escape from Outworld stored its level and object data inside JSON files. The RapidJSON library was utilized to get deserialization up and running as quickly as possible. The JSON files were passed to a massive factory system that read the file, allocated the required resources, and returned a working object. This allowed levels to easily create objects without bloating compile time with header includes.
Levels were designed in Tiled, an open-source 2D level creation tool. The levels were exported as JSON files and read into our game. This caused a few blocks during development. The JSON format did not play nice with Tiled, and occasionally certain information was omitted during the saving process. To mitigate this, tileset and object data were stored externally in other JSON files. The level creation system would look at the name of the tileset being used and check for a tileset file of the same name. As a bonus, this allowed tilesets to be updated quickly without re-exporting the level.

I have had my hand in quite a few custom engines. Squarepunch, Cleanup on Aisle 6.67, and EFO: Escape from Outworld all used custom engines. Across these projects, I have learned multiple different ways to organize game systems, objects, components, and memory. Over the summer of 2022, I decided to create a small custom engine to experiment with split-screen rendering and input. I also used this project to try and familiarize myself with dynamically linked libraries. Each game system was split into its own library.
As with all first attempts, the result was rough, but I learned a wealth of information about how to set up input systems for multiple controllers and exactly when and where you should use a DLL.
