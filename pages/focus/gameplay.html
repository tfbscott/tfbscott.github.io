Gameplay Programming

EFO: Escape from Outworld features a mechanic where the player can pick up and move blocks to solve puzzles. Since EFO needed to be up and playable in just 7 weeks, the design team was rapidly prototyping while the core engine was being built. To accommodate this, the act of picking up and moving an object was split into two components, Grabber and Handle. The grabber would always be set on our Mouse object and controlled the speed at which objects moved toward its center and the max number of objects it could grab. The Handle was used as a flag so the Grabber knew what it could grab. This allowed the design team to easily flag an object as grabbable and experiment with object movement speeds and grabbing multiple objects.

Early in the development of P.U.P.P.I.T. the idea was pitched to allow the player to swap between multiple guns. To accommodate this, guns were created as their own actors and forcefully bound to the player when the level loaded. The gun actors were stored in an array and updated during the Players update loop. An index was used to identify what the currently attached gun was. Each gun featured a state machine with 5 states: Equip, Idle, Fire, Reload, and DeEquip. Entry to certain states would be denied based on the gun's current values, if the gun's ammunition counter was zero, the Fire state would be denied.
