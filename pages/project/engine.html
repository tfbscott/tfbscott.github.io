CUSTOM ENGINE
C++ | 2022

Over the summer of 2022, I created an Engine Component System Engine using the skills I have aquired across my 3 years at Digipen. This project was my first real experience with OpenGL and creating multiple DLLs for a project.
The engine features:

  • A core system that handles the creation of Game Objects and updating of Systems
  • A 3D renderer utilizing OpenGL and GLFW that supports splitscreen
  • An Input Manager that supports multiple input devices
  • A simple Kinematics System that updates object positions and resolves AABB collision
  • A simple behavior system that is used to control objects on screen

Core

The Engine Core handles the creation of GameObjects, Components, and Systems. It is designed to be as multi-platform as possible. Each component and system would derive from template classes defined in Core.h, and can then be added to the list of system the Core's Update function runs through each frame.
User Input is partually managed by the Engine Core. The system for registering key presses to actions and broadcasting action events exists, but translating data from actual input devives is not set up. This is left to other libraries (In this case, GLFW).
By default, a systems update loop is parallel. This improves performance by updating multiple components at the same time on multiple threads. This feature is NOT used often due to bugs with components accessing outside data.

Renderer

The renderer uses GLFW to draw objects on screen. Shape and Texture classes hold information for verticies and color data and can be shared between Renderer Components. Each frame, the Render System grabs all active cameras, and creates a viewport based off of the width, height, x, and y set by the Camera component. Each Renderer Component is then drawn onto each viewport.

Kinematics

To allow simple object movement, a barebones kinematics system was created. The Kinematics Component accuratly updates an objects Transformation using velocity and acceleration.
An Axis Aligned Bounding Box collision system was also created. This system allows for collision between multiple bounding boxes, as well as the ability to send a raycast out into the enviroment and return the firstmost collision.
The Kinematics System is currently the only system that updates components on multiple threads. There are plans to update the collision system to do this as well, but that is a larger undertaking.

Behavior

The behavior system is simply a way for a user (mostly me) to create per object behaviors. The Base Behavior Component provides OnStart, OnUpdate, and OnDelete functions for child classes to override. Functions to subscribe Action Events are also provided to ease the process.
An On Collision Event function is provided as well, and is pre-subscribed to a Collision Event that is provided in the Engine Core. Theoretically, any Collision system provided will throw a Collision Event when colliding, and this function will be called.

Future

This project was an amazing learning experience for me, and I plan to continue updating the systems as I learn more. Certain ideas for this engine were scrapped due to complexity or requiring major refactoring.
